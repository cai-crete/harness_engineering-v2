## 브랜드 원칙

- 전문적이고 간결 — 설명하지 않고 선언한다
- 기술 용어와 설계 언어 혼용
- 한국어 + 영어 병용 (기술 레이어: 영어, 개념 레이어: 한국어)
- 불필요한 수식어 없음, 직관적 명명
- 노드별 브랜드 가이드라인: `project_[node_name]/_context/brand-guidelines.md` 참조

## 네이밍 컨벤션

- 노드명: `동사 to 명사` 형식 (예: `sketch to plan`, `change viewpoint`)
- 버전: `v[N]` (예: `v4`)
- 파일: `kebab-case`
- 컴포넌트: `PascalCase`

## UI 원칙

- 노드 앱은 단일 목적 UI — 하나의 노드, 하나의 기능
- 입력 → 실행 → 출력의 3단계 흐름이 명확해야 함
- 결과물에는 항상 "참고용 시각화" 워터마크 자동 삽입
- 법규·구조 보장 문구 전면 배제

## 컬러 & 타이포그래피 등 UI 스타일링
