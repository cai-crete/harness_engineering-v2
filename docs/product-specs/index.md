# Product Specs — 노드 스펙 인덱스

각 노드의 입력·출력 계약, Protocol 버전, 컴플라이언스 체크리스트를 정의합니다.

| 노드 | 스펙 파일 | 상태 |
|------|-----------|------|
| (각 노드) | `node-spec-template.md` 기반으로 작성 | 미작성 |

---

## 결과물 _template 파일 추적

각 노드는 생성하는 결과물(아티팩트)의 구조·포맷 계약을 `_template` 파일로 정의합니다.
노드 실행 시 이 파일을 로드하여 실제 아티팩트 생성의 기준으로 사용합니다.
네이밍 컨벤션: `[node_name]-[output-name]_template.md` (또는 `.txt`)

| 노드 | 결과물 | _template 파일 | 상태 |
|------|--------|----------------|------|
| (각 노드) | product-spec에 정의 | — (노드 스펙 확정 후 작성) | 미작성 |

> 노드 스펙이 확정되면 해당 행을 실제 결과물명과 파일 경로로 채웁니다.
> 상태: `미작성` / `작성 완료`

---

## 신규 노드 스펙 작성 시

`node-spec-template.md`를 복사하여 `[node_name].md`로 저장합니다.

---

## 노드 전용 실패 모드 등록 방법

Stage B에서 노드 고유의 실패 패턴이 발견되면 해당 노드의 스펙 파일(`[node_name].md`)의 `## 알려진 실패 패턴` 섹션에 기록합니다.

```markdown
| [패턴명] | [재현 조건] | [Protocol 수정 방향] |
```

공통 패턴으로 확산될 경우 `docs/RELIABILITY.md`의 공통 실패 모드 테이블에 추가합니다.
