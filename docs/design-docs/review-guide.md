# Review Guide — 오류 검토 보고서 작성 표준

> 이 가이드는 하네스 및 개발 산출물의 정합성 검토 세션에서
> **오류 검토 보고서를 어떻게 작성하고 진행하는가**를 정의합니다.

---

## 1. 보고서 작성 트리거

다음 상황에서 오류 검토 보고서를 작성합니다:

| 상황 | 예시 |
|------|------|
| 하네스 초기 구성 완료 후 | 신규 노드 개발 전 환경 세팅 완료 시 |
| Protocol 버전 업 후 | v1 → v2 전환 전 |
| 사용자가 명시적으로 검토를 요청할 때 | "정합성 검토 해줘" |
| 주요 하네스 파일 다수 수정 후 | 3개 이상 파일 동시 변경 시 |

---

## 2. 보고서 3단계 프로세스

### Step 1 — 검토 및 보고서 작성

모든 대상 파일을 읽고 아래 세 축에서 검토한다:
- **구조 정합성**: 파일 간 참조·네이밍·섹션 구조가 일관되는가
- **계약 일관성**: Input/Output Contract, API 아키텍처, 모드별 처리가 명확히 정의됐는가
- **운영 안전성**: 훅·패턴·에이전트 경계가 의도대로 동작하는가

보고서 본문에 결함을 기술한 뒤, **제일 하단에 체크리스트를 배치**한다.

### Step 2 — 사용자 승인 후 수정 진행

보고서 작성 후 사용자 승인을 받은 뒤 수정을 진행한다.
승인 없이 파일을 수정하지 않는다.

### Step 3 — 체크리스트 체크 + 해결 방법 기록

수정 완료 후 체크리스트의 해당 항목에 체크하고,
체크리스트 항목 **아래에 해결 방법을 설명**한다.

---

## 3. 보고서 포맷

### 파일명 규칙

```
docs/reviews/Error_Review_Report-v.[N]-[YYMMDD].md
```

| 토큰 | 규칙 |
|------|------|
| `[N]` | 1부터 시작하는 정수. 동일 프로젝트 내 누적 버전. |
| `[YYMMDD]` | 작성일. 연도 2자리 + 월 2자리 + 일 2자리. 예: 2026년 4월 14일 → `260414` |

### 보고서 본문 구조

```markdown
# Error Review Report — v.[N]

> 작성일: YYYY-MM-DD
> 검토 범위: [검토 대상 요약]
> 검토자 역할: 시니어 개발 전문가 (독립 정합성 검토)

---

## 개요
[검토 대상 파일 목록 및 작업 유형]

---

## 결함 목록

### [SEVERITY-N] [결함 제목]
**파일**: [파일 경로]
**위치**: [섹션 또는 줄]
**현상**: [무엇이 잘못됐는가]
**영향**: [방치 시 어떤 문제가 발생하는가]
**처방**: [구체적 수정 방향]

---

## 결함 요약
[심각도별 집계 테이블]

---

## 긍정 항목 (잘 된 것)
[결함이 아니지만 명시적으로 기록할 올바른 결정들]

---

## 권고 사항
[수정 우선순위 분류]

---

## 전체 판정
[CRITICAL/HIGH/MEDIUM/LOW 건수 + 하네스 구성 완결성 판정]

---

## 수정 체크리스트

- [ ] [SEVERITY-N] [결함 제목] — [파일명]
- [ ] [SEVERITY-N] [결함 제목] — [파일명]
...
```

---

## 4. 심각도 기준

| 심각도 | 정의 | 수정 시점 |
|--------|------|----------|
| **CRITICAL** | 런타임 오류 또는 데이터 손실 유발 가능 | 즉시. 다른 작업 중단 |
| **HIGH** | 의도치 않은 에이전트 행동 유발. Stage B 진입 불가 | Stage B 진입 전 필수 |
| **MEDIUM** | 계약 누락으로 구현 시 오판 가능 | 개발 착수 전 권고 |
| **LOW** | 표기 오류, 의미 모호, 인덱스 불일치 | 다음 버전 반영 가능 |
| **ADVISORY** | 결함은 아니나 개선 여지 있음 | 선택적 |

---

## 5. 아카이브 규칙

- 보고서는 `docs/reviews/` 에 버전별로 누적 보관한다.
- 보고서를 삭제하거나 덮어쓰지 않는다.
- 동일 날짜에 2번째 보고서가 필요한 경우: `v.[N]-[YYMMDD]-2.md` 로 구분한다.
- `docs/reviews/` 폴더는 에이전트가 임의로 재구성하지 않는다.
