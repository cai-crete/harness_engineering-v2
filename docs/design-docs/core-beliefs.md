# Core Beliefs — 노드 출력 기준

> 이 파일은 각 노드의 결과물이 어떤 기준으로 만들어지는지를 정의합니다.
> Agent가 구현 방향을 결정할 때, 또는 기술적으로 동등한 선택지 앞에서
> "이 방향이 우리 기준에 맞는가"를 판단하는 근거입니다.
> Protocol이 이 원칙들과 충돌하면, Protocol을 수정합니다.

---

## 1. Protocol-First (출력 품질의 천장)

**출력물의 품질 상한은 Protocol이 결정합니다. 코드는 품질을 올릴 수 없습니다.**

코드가 아무리 완벽해도 Protocol이 모호하면 출력물이 기복을 가집니다.
반대로 Protocol이 정밀하면 평범한 코드도 일관된 품질의 출력물을 만듭니다.

→ 출력 품질 문제가 발생하면 코드가 아닌 Protocol을 먼저 점검합니다.
→ Protocol을 수정하지 않은 채 코드로 출력 결과를 조작하지 않습니다.

---

## 2. Physical Reality Principle (출력물의 본질)

**출력물은 완성된 물리적 현실의 재현이지, 상상의 창작이 아닙니다.**

모든 노드에서 AI의 역할은:
- 자유롭게 창작하는 것 ❌
- 존재하는 현실을 분석하고 재렌더링하는 것 ✅

입력 데이터는 "참고 자료"가 아니라 **"완성된 현실"**입니다.
출력물은 그 현실을 다른 형식·관점·추상 수준으로 변환한 결과입니다.
입력에 없는 요소를 출력물에 추가하는 것은 오염입니다.

---

## 3. Geometry Sanctuary (출력물의 불변 경계)

**건물의 비율, 구조, 형태는 출력물에서 절대 변경되지 않습니다.**

출력물에서 변경 가능한 것:
- 각 노드가 product-spec에서 명시적으로 허용한 속성
- 표현 방식 (시점, 스타일, 매체 등 노드 목적에 따라 정의)

출력물에서 변경 불가능한 것:
- 건물의 높이·폭·비율
- 구조적 요소 (기둥, 슬래브, 개구부)
- 공간 배치 및 동선 체계

**기하학이 변형된 출력물은 오염(Contamination)으로 즉시 판정합니다.**

---

## 4. Concrete Specification (출력물의 실행 단위)

**출력물은 반드시 구체 파라미터로 실행된 결과여야 합니다.**

추상적 명령("간결하게", "자연스럽게", "강조해줘")은 의도를 담고 있지만,
AI의 실행 단위는 구체적이고 측정 가능한 파라미터입니다.
추상 명령이 구체 파라미터로 변환되지 않은 출력물은 재현 불가능하며 품질을 보장할 수 없습니다.

| 노드 유형 | 추상 명령 | 구체 파라미터 |
|-----------|----------|--------------|
| 텍스트 생성 | "간결하게 써줘" | 목표 단어 수, 문단 수, 문장 길이 상한 |
| 다이어그램 | "동선을 강조해줘" | 강조 대상 경로, 두께·색상 가중치 수치 |
| 스타일 편집 | "따뜻한 느낌으로" | 색온도(K), 소재 카테고리, 조명 방향 |
| 시각화 공통 | "자연스럽게" | 해당 노드의 품질 기준 수치로 변환 |

추상 언어가 구체 파라미터로 변환되지 않으면 할루시네이션의 주요 경로가 됩니다.

---

## 5. Boundary Inference (출력물의 완결 의무)

**입력 범위를 초과하는 요청이 와도 출력물은 완결되어야 합니다.**

입력에서 명시되지 않은 영역을 요청받았을 때 출력물이 취할 수 있는 형태:

- 원본 입력 그대로 반환 ❌
- "처리 불가" 응답 ❌
- 입력에서 확인 가능한 패턴·맥락·논리를 기반으로 추론하여 완성 ✅

추론의 근거는 항상 입력 데이터 내에 존재해야 합니다. 근거 없는 창작은 오염입니다.
출력물은 추론 과정과 근거를 설명문에 명시해야 합니다.

| 노드 유형 | 경계 상황 | 추론 근거 |
|-----------|----------|----------|
| 텍스트 생성 | 특정 섹션 정보 누락 | 문서 내 다른 섹션의 맥락·어조·구조 |
| 도면 변환 | 일부 치수 미기재 | 기재된 치수의 비례 관계 |
| 다이어그램 | 동선 일부 불명확 | 공간 배치와 프로그램 논리 |
| 시각화 공통 | 요청 영역이 입력에 미노출 | 노출 영역에서 확인된 구조·재료·비례 패턴 |

---

## 6. Harness Engineering (출력 일관성의 보장)

**세션이 바뀌어도 출력 기준은 변하지 않습니다. 하네스가 그것을 보장합니다.**

AI 세션은 무상태(stateless)입니다. 하네스 없이는 매 세션마다 출력 기준이 달라집니다.

- `_context/` 파일들이 노드 수준 출력 기준 보관소
- `CAI/docs/` 파일들이 회사 수준 출력 기준 보관소
- Protocol 파일이 노드 출력의 실행 기준

하네스 품질이 기능 품질보다 낮으면 매 세션마다 같은 편차가 반복됩니다.
**하네스 품질 = 출력 일관성**이 CAI의 개발 원칙입니다.
