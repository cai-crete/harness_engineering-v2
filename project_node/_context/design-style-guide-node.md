# CAI CANVAS 앱 인터페이스 디자인 시스템

본 문서는 CAI CANVAS 서비스의 일관된 브랜드 정체성과 전문가급 UX를 보장하기 위한 인터페이스 디자인 표준입니다.

**문서 구성:**
- **PART A** : @design-style-guide-CAI.md — 앱 인터페이스 공통 디자인 시스템 (모든 CAI 노드에 적용)
- **PART B** : @design-style-guide-node.md — 'node' 노드 전용 문서 템플릿 디자인 시스템

---

# PART B · 'node' 노드 — 문서 템플릿 디자인 시스템

> 이 파트의 모든 내용은 **'node' 노드 전용**입니다.
> 사용자가 Agent에게 입력한 데이터(와이어프레임, 텍스트, 정보구조도, 플로우차트 등)와 PART A (@design-style-guide-CAI.md)를 기반으로 작성되는 문서입니다.
> PART A의 앱 인터페이스 디자인 규칙을 기준으로 관리되며, 예외사항이 있을 경우 별도 표기합니다.
> 예외사항이 작성될 경우 해당 사항이 PART A보다 우선시됩니다.
---

## B.0 'node' 서비스 개요

> /shape 실행 후 채울 것: 노드 목적, 주요 사용자, 작업 컨텍스트

---

## B.1 폰트 예외사항

> 없으면 이 섹션 생략. 있을 경우: PART A §A.1 기준에서 변경되는 폰트와 사유 명시.

---

## B.2 컬러 예외사항

> 없으면 이 섹션 생략. 있을 경우: PART A §A.3 기준에서 추가되는 색상과 사용 맥락 명시.

---

## B.3 레이아웃 특이사항

> /shape 실행 후 채울 것: 좌측 툴바 버튼 구성 (§A.6.1 참조), 노드 고유 UI 컴포넌트

---

## B.4 상태 정의

> /shape 실행 후 채울 것: 입력 / 로딩 / 완료 / 오류 각 상태의 UI 요구사항

## B.5 워터마크 규칙
