## Design Context

### Users
건축 설계 전문가 (건축사, 실내건축사, 건설사 담당자).
빠른 설계 의사결정 보조 도구로 사용. 복잡한 기능보다 속도·신뢰성을 우선시함.
데스크탑 환경에서 다른 설계 툴과 병행 사용.

### Brand Personality
전문적 · 간결 · 기술적 (Professional, Concise, Technical)
"설명하지 않고 선언한다." 불필요한 수식어 없음.
한국어 + 영어 병용 (기술 레이어: 영어, 개념 레이어: 한국어).

### Aesthetic Direction
흑백 미니멀리즘. 컬러 없음 (오류 red 유일한 예외).
Bebas Neue (헤더·CTA) + Pretendard (본문) 2종 폰트 고정.
Light 모드. 흰 배경 + 검정 텍스트. 그레이스케일 7색 (#000, #333, #666, #999, #CCC, #EEE, #FFF).
레퍼런스: 고급 건축 출판물, 기술 도면 인터페이스.

### Design Principles
1. **단일 목적 UI**: 노드당 하나의 기능. 입력→실행→출력 3단계 플로우만.
2. **계층 준수**: CTA-primary는 화면당 1개. Tier 1/2/3 위계 엄수 (design-style-guide-CAI.md §A.0 참조).
3. **토큰 외 수치 금지**: 색상·간격·곡률은 반드시 설계된 토큰 사용 (임의 수치 금지).
4. **워터마크 필수**: 모든 결과물에 "참고용 시각화" 자동 삽입.
5. **법규·구조 보장 문구 전면 배제**: "준수", "보장", "적합" 등의 표현 금지.

### Constraints (CAI 시스템 — impeccable 기본값 오버라이드)
- 색상: Hex 그레이스케일 7색만. OKLCH·color-mix 사용 금지.
    black #000000 / white #FFFFFF / gray-500 #333333 / gray-400 #666666
    gray-300 #999999 / gray-200 #CCCCCC / gray-100 #EEEEEE
    예외: 오류 Toast에만 red 허용.
- 폰트: Bebas Neue (헤더/CTA) + Pretendard (본문) 고정. 다른 폰트 사용 금지.
- 간격: 8px 그리드 토큰만 사용.
    space-0: 0 / space-1: 8px / space-2: 16px / space-3: 20px
    space-4: 32px / space-5: 40px / space-6: 48px / space-8: 64px
- 곡률: radius-box(10px) 또는 radius-pill(5rem) 2종만.
- 그림자: shadow-float (0 10px 15px -3px rgba(0,0,0,0.1)) 단일 적용.
- 높이: CTA-primary/secondary 52px / 기타 인터랙티브 최솟값 44px.
- 다크모드: 미지원. 흰 배경 고정.
